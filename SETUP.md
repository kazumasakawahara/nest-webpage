# はじめてのセットアップ手順書（Mac 用）

このサイト（NPO法人 nest 公式サイト）を、**別の Mac で動かして編集できるようにする**ための手順書です。
プログラミングがはじめての方でも、上から順番にコピー＆ペーストしていけば動くように書いています。

> 📌 この手順でできるようになること
> - GitHub からサイトのデータを自分の Mac にコピーする
> - 自分の Mac でサイトを表示して、文章・お知らせ・画像を編集する
> - 編集した結果をブラウザで確認する
>
> 会員専用エリア（`/members/`）を動かす方法は、最後の「上級編」で概要だけ触れます。

---

## 0. 用語ミニ辞典（読み飛ばしてOK）

| 言葉 | かんたんな意味 |
|------|----------------|
| **リポジトリ（repository）** | プロジェクトのデータ一式が入った「フォルダ」のこと。GitHub 上にあります。 |
| **clone（クローン）** | GitHub 上のリポジトリを、自分の Mac に丸ごとコピーすること。 |
| **ターミナル（Terminal）** | 文字でコマンドを打って Mac を操作するアプリ。Mac に最初から入っています。 |
| **Node.js（ノード）** | このサイトを動かすために必要なソフト。 |
| **npm（エヌピーエム）** | Node.js に付いてくる「部品をまとめてインストールする道具」。 |
| **開発サーバ** | 編集中のサイトを自分の Mac の中だけで表示する仕組み。`http://localhost:4321` で見られます。 |

## 1. 準備するもの（最初の1回だけ）

### 1-1. ターミナルを開く

`command（⌘）` キー + `スペース` を押して **Spotlight** を出し、「ターミナル」と入力して `Enter`。
黒い（または白い）文字入力の画面が出ます。これがターミナルです。

> 💡 この手順書に出てくる「ターミナルで実行」は、**コマンドをコピーして貼り付け → `Enter` を押す**という意味です。

### 1-2. Git が入っているか確認する

ターミナルで次を実行：

```sh
git --version
```

- `git version 2.xx.x` のように表示されれば **OK**。次（1-3）へ進んでください。
- `xcode-select` のインストールを促す画面が出たら、画面の指示に従って「インストール」を押し、完了してからもう一度上のコマンドを実行してください。

### 1-3. Node.js をインストールする

このサイトは **Node.js のバージョン 22.12.0 以上** が必要です。

**いちばんかんたんな方法（おすすめ）：**

1. ブラウザで [https://nodejs.org/](https://nodejs.org/) を開く
2. 「**LTS**」と書かれた左側のボタン（安定版）をクリックしてダウンロード
3. ダウンロードした `.pkg` ファイルをダブルクリックし、「続ける」を押していくだけでインストール完了

インストールできたか、ターミナルで確認：

```sh
node -v
```

`v22.12.0` 以上（例：`v22.14.0` や `v24.x.x`）が表示されれば **OK** です。

> ⚠️ もし `v20.x.x` など古い番号が出たら、上のサイトから新しい LTS を入れ直してください。

### 1-4. エディタ（VS Code）を用意する ※推奨

文章やコードを編集するためのアプリです。なくても進められますが、あると圧倒的に楽です。

- [https://code.visualstudio.com/](https://code.visualstudio.com/) からダウンロードしてインストール。

## 2. サイトのデータを自分の Mac にコピーする（clone）

### 2-1. 置き場所を決める

ここでは例として、ホーム直下に `Projects` フォルダを作って、その中に置きます。

```sh
mkdir -p ~/Projects
cd ~/Projects
```

> 💡 `cd` は「指定したフォルダに移動する」コマンドです。

### 2-2. clone する

このリポジトリは**公開**なので、ログインなしでコピーできます。

```sh
git clone https://github.com/kazumasakawahara/nest-webpage.git
```

完了したら、コピーされたフォルダの中に入ります：

```sh
cd nest-webpage
```

> ✅ ここまでで、`~/Projects/nest-webpage` にサイト一式が入りました。

---

## 3. 必要な部品をインストールする

フォルダの中（`nest-webpage`）にいる状態で、次を実行：

```sh
npm install
```

- インターネットから部品をダウンロードするので、**数分かかる**ことがあります。
- 途中で `warn`（警告）がたくさん出ても、最後にエラーで止まらなければ大丈夫です。

> 💡 これは最初の1回だけでOKです（部品が増えたときだけ再実行）。

---

## 4. サイトを表示してみる（開発サーバの起動）

```sh
npm run dev
```

ターミナルに次のような表示が出ます：

```
  astro  v6.x.x ready in xxx ms
  ┃ Local    http://localhost:4321/
```

ブラウザ（Safari や Chrome）のアドレス欄に **`http://localhost:4321`** と入力して `Enter`。
サイトが表示されれば成功です！🎉

> 💡 **止め方**：ターミナルで `control（^）` + `C` を押すとサーバが止まります。
> もう一度見たくなったら、`nest-webpage` フォルダで `npm run dev` を実行すればOK。

---

## 5. 編集してみる

`npm run dev` を動かしたまま、別ウィンドウでファイルを編集すると、**保存した瞬間にブラウザが自動で更新**されます。

| やりたいこと | 編集する場所 |
|--------------|--------------|
| ページの文章を変える | `src/pages/` の中の `.astro` ファイル |
| 「お知らせ」を追加する | `src/content/news/` に新しい `.md` ファイルを作る |
| サイト名・ナビ・各種数値 | `src/lib/site.ts` |
| 色やフォントなどデザイン | `src/styles/global.css` |
| 写真・画像を差し替える | `public/images/`（詳細は `public/images/README.md`） |

### お知らせを1件追加してみる例

`src/content/news/` の中に `2026-06-10-test.md` のようなファイルを作り、次を貼り付けます：

```md
---
title: テストのお知らせ
date: 2026-06-10
tag: お知らせ
summary: 一覧に出る短い要約（任意）
---

ここに本文を書きます。**強調**や [リンク](/contact/) も使えます。
```

保存してブラウザを見ると、お知らせ一覧に反映されます。

> 📝 ファイル名は `YYYY-MM-DD-スラッグ.md`（例：`2026-06-10-test.md`）の形がおすすめです。
> `tag` は `お知らせ / イベント / 事業所 / メディア` のいずれかにしてください。

## 6. 本番用に書き出して確認する（任意）

実際に公開されるときと同じ形（静的ファイル）に書き出して確認できます。

```sh
npm run build      # dist/ フォルダに完成版を書き出す
npm run preview    # 書き出した完成版をブラウザで確認
```

`build` がエラーなく終われば、公開できる状態になっています。

## 7. 困ったときは（トラブル対処）

| 症状 | 対処 |
|------|------|
| `command not found: npm` | Node.js が入っていません。「1-3」をやり直してください。 |
| `command not found: git` | 「1-2」をやり直してください。 |
| `npm run dev` でポートが使えないと出る | すでに別で起動中かも。ターミナルで `control + C` で止めてから、もう一度実行。 |
| `npm install` が赤いエラーで止まる | Node のバージョンが古い可能性。`node -v` で 22.12.0 以上か確認。 |
| ブラウザに何も出ない | アドレスが `http://localhost:4321` になっているか確認。`npm run dev` が動いたままか確認。 |
| そもそも今どこのフォルダにいるか不安 | `pwd` で現在地を確認。末尾が `/nest-webpage` ならOK。違えば `cd ~/Projects/nest-webpage`。 |

## 8. 上級編：会員専用エリア（/members/）を動かしたい場合

公開ページの編集だけなら、ここは不要です。

`/members/` 以下は**ログイン認証付き**で、外部サービス（Supabase / Resend）の鍵が必要です。
鍵は GitHub には含まれていません（セキュリティのため）。動かすには：

1. リポジトリ直下の `.env.example` を `.env.local` という名前でコピー
   ```sh
   cp .env.example .env.local
   ```
2. `.env.local` を開き、**Supabase / Resend の認証情報**を記入（管理者の河原さんから受け取ってください）
3. データベースの初期設定は `db/README.md` の手順に従う
4. その後 `npm run dev` で会員エリアも動作します

詳しい運用は次のドキュメントを参照：
- 設計仕様：`HANDOVER-PHASE1.md`
- デプロイ手順：`docs/operations/deployment.md`
- 会員管理の運用：`docs/operations/member-management.md`

## 9. 編集した内容を GitHub に戻したい場合（概要）

自分の Mac で編集しただけでは、GitHub 上のサイトは変わりません。
変更を GitHub に反映（push）するには、**リポジトリへの書き込み権限**が必要です（管理者の河原さんに依頼してください）。

権限がある場合の基本の流れ（参考）：

```sh
git checkout -b my-edit       # 作業用のブランチを作る
git add -A                    # 変更をまとめる
git commit -m "お知らせを更新"   # 変更を記録する
git push -u origin my-edit    # GitHub に送る
```

その後、GitHub 上で「Pull Request」を作成し、河原さんに確認してもらいます。
