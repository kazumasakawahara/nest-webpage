NPO法人 nest — Web Setup Guide

はじめてのセットアップ手順書

別の Mac でこのサイトを動かし、編集できるようにするための手順です。
上から順番にコピー＆ペーストすれば大丈夫。プログラミング未経験でも進められます。

📌 この手順でできるようになること

  • GitHub からサイトのデータを自分の Mac にコピーする
  • 自分の Mac でサイトを表示して、文章・お知らせ・画像を編集する
  • 編集した結果をブラウザで確認する

会員専用エリア（/members/）を動かす方法は、最後の「上級編」で概要だけ触れます。公開ページの編集だけなら不要です。

用語ミニ辞典（読み飛ばしてOK）

言葉かんたんな意味
リポジトリプロジェクトのデータ一式が入った「フォルダ」。GitHub 上にあります。
cloneGitHub 上のリポジトリを、自分の Mac に丸ごとコピーすること。
ターミナル文字でコマンドを打って Mac を操作するアプリ。最初から入っています。
Node.jsこのサイトを動かすために必要なソフト。
npmNode.js に付いてくる「部品をまとめてインストールする道具」。

1

準備するもの（最初の1回だけ）

1-1. ターミナルを開く

⌘ + スペース で Spotlight を出し、「ターミナル」と入力して Enter。文字入力の画面が出ればOKです。

💡 ヒント：以下の「ターミナルで実行」は、コマンドをコピーして貼り付け → Enter という意味です。各コマンド右上の「コピー」ボタンが使えます。

1-2. Git が入っているか確認

$ git --version

git version 2.xx.x と出れば OK。インストールを促す画面が出たら、指示に従って「インストール」してから再実行してください。

1-3. Node.js をインストール（バージョン 22.12.0 以上が必須）

いちばんかんたんな方法：

  1. ブラウザで https://nodejs.org/ を開く
  2. 「LTS」と書かれた安定版ボタンをクリックしてダウンロード
  3. ダウンロードした .pkg をダブルクリックし、「続ける」を押すだけ

入ったか確認：

$ node -v
⚠️ 注意：v22.12.0 以上（例 v22.14.0 / v24.x.x）が出ればOK。v20 など古い番号なら入れ直してください。

1-4. エディタ（VS Code）※推奨

文章やコードの編集が楽になります。code.visualstudio.com からどうぞ。

2

データを自分の Mac にコピーする（clone）

2-1. 置き場所を決める

例として、ホーム直下に Projects フォルダを作ってその中へ：

$ mkdir -p ~/Projects $ cd ~/Projects

2-2. clone する

このリポジトリは公開なので、ログインなしでコピーできます。

$ git clone https://github.com/kazumasakawahara/nest-webpage.git $ cd nest-webpage
✅ 完了：~/Projects/nest-webpage にサイト一式が入りました。
3

必要な部品をインストールする

nest-webpage フォルダの中にいる状態で：

$ npm install
💡 ヒント：数分かかることがあります。warn（警告）が出ても、最後にエラーで止まらなければ大丈夫。最初の1回だけでOKです。
4

サイトを表示してみる（開発サーバ）

$ npm run dev

次のような表示が出ます：

astro v6.x.x ready in xxx ms ┃ Local http://localhost:4321/

ブラウザのアドレス欄に http://localhost:4321 と入力して Enter。サイトが表示されたら成功です！🎉

💡 止め方：ターミナルで control（^） + C。再開は npm run dev をもう一度。
5

編集してみる

npm run dev を動かしたままファイルを編集して保存すると、ブラウザが自動で更新されます。

やりたいこと編集する場所
ページの文章を変えるsrc/pages/ の中の .astro
「お知らせ」を追加src/content/news/ に .md を新規作成
サイト名・ナビ・数値src/lib/site.ts
色やフォント（デザイン）src/styles/global.css
写真・画像の差し替えpublic/images/

お知らせを1件追加する例

src/content/news/ に 2026-06-10-test.md を作り、以下を貼り付けます：

--- title: テストのお知らせ date: 2026-06-10 tag: お知らせ summary: 一覧に出る短い要約（任意） --- ここに本文を書きます。**強調**や [リンク](/contact/) も使えます。
📝 メモ：ファイル名は YYYY-MM-DD-スラッグ.md 推奨。tag は お知らせ / イベント / 事業所 / メディア のいずれか。
6

本番用に書き出して確認（任意）

$ npm run build # dist/ に完成版を書き出す $ npm run preview # 完成版をブラウザで確認

build がエラーなく終われば、公開できる状態です。

上級編：会員専用エリアを動かす場合

公開ページの編集だけなら不要です。/members/ はログイン認証付きで、外部サービス（Supabase / Resend）の鍵が必要です。鍵は GitHub には含まれていません。

  1. .env.example を .env.local という名前でコピー：
    $ cp .env.example .env.local
  2. .env.local を開き、Supabase / Resend の認証情報を記入（管理者の河原さんから受け取る）
  3. データベースの初期設定は db/README.md の手順に従う
  4. その後 npm run dev で会員エリアも動作します
📚 詳細：HANDOVER-PHASE1.md ／ docs/operations/deployment.md ／ docs/operations/member-management.md
9

編集を GitHub に戻したい場合（概要）

自分の Mac で編集しただけでは、GitHub 上のサイトは変わりません。反映（push）にはリポジトリへの書き込み権限が必要です（管理者の河原さんに依頼）。権限がある場合の基本の流れ：

$ git checkout -b my-edit # 作業用ブランチを作る $ git add -A # 変更をまとめる $ git commit -m "お知らせを更新" # 変更を記録 $ git push -u origin my-edit # GitHub に送る

その後、GitHub 上で「Pull Request」を作成し、河原さんに確認してもらいます。
