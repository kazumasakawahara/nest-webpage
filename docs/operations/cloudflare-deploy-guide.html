DEPLOY GUIDE — 特定非営利活動法人 nest

このサイトを Cloudflare で
法人ドメインに公開する手順

ローカルで作成中のサイトを、インターネットに公開するまでの全工程

公開先（予定）：www.nponest.org

このサイトの仕組み（最初に知っておくこと）

このサイトは Astro という仕組みで作られ、公開先として Cloudflare を使う設定が既に入っています（astro.config.mjs に www.nponest.org も設定済み）。

サイトは2種類のページでできています：

  • 公開ページ（トップ・親なき後・各事業所など）… 誰でも見られる。動作が軽く、設定だけで公開できる。
  • 会員ページ（/members/）… ログインが必要。Supabase という外部サービスと「鍵（パスワード的な情報）」が必要で、これを Cloudflare に登録しないと動きません。
⚠ 重要な分かれ道 会員ページを今すぐ動かすには「鍵」の登録（Step 4）が必須です。まず公開ページだけ先に出して、会員ページは後日にする選択もできます。急がないなら後者がおすすめ（準備が少なくて済みます）。

全体の流れ

① ドメイン準備 → ② GitHub連携 → ③ ビルド設定 → ④ 鍵の登録 → ⑤ ドメイン割当 → ⑥ 確認

※ ④は会員ページを動かす場合のみ必須。公開ページだけなら飛ばせます。

1

ドメイン nponest.org を Cloudflare に載せるDNS を Cloudflare に管理させる

独自ドメインで公開するには、まず nponest.org の「住所管理（DNS）」を Cloudflare に任せます。今の状況で2つに分かれます。

A. nponest.org をまだ取得していない場合
Cloudflare の 「Registrar（ドメイン登録）」で nponest.org を新規取得します。取得と同時に自動で Cloudflare 管理になるので、一番シンプルです（料金は実費のみ・上乗せなし）。
B. すでに他社で取得済みの場合（お名前.com・ムームードメイン・Xserver など）
① Cloudflare で「Websites → Add a site」から nponest.org を追加 → ② Cloudflare が示す2つのネームサーバーを、今の登録業者の管理画面で設定（ネームサーバー変更）→ ③ 反映を待つ（数時間〜最大48時間）。
先に確認 nponest.org が「誰の名義で・どこの業者で登録されているか」を把握しておいてください。法人名義の管理情報（ログイン）が必要になります。
2

サイトと Cloudflare をつなぐGitHub のコードを自動で公開する設定

  1. Cloudflare ダッシュボード → 「Workers & Pages」→「Create」→「Pages」→「Connect to Git」
  2. GitHub へのアクセスを許可し、リポジトリ kazumasakawahara/nest-webpage を選ぶ
  3. 公開の元にするブランチ＝ main を指定（最新版は既にここに入っています）
この設定の良いところ 一度つなげば、今後 main が更新されるたびに自動で公開サイトも更新されます。手作業のアップロードは不要です。
3

ビルド設定を入力するサイトの組み立て方を Cloudflare に教える

連携時の設定画面で、以下をそのまま入力します。

項目入力する値
フレームワーク（Framework preset）Astro
ビルドコマンド（Build command）npm run build
出力フォルダ（Build output directory）dist

※ もし「Node のバージョン」で失敗したら、環境変数に NODE_VERSION ＝ 20 を追加してください。

4

鍵（環境変数）を登録する会員ページを動かす場合のみ必須

「Settings → Environment variables → Production」に、ローカルの .env と同じ値を登録します。

名前（キー）種別
SUPABASE_URL通常
SUPABASE_ANON_KEY通常
SUPABASE_SERVICE_ROLE_KEY秘密（暗号化）
RESEND_API_KEY秘密（暗号化）
MAIL_FROM通常
PUBLIC_SITE_URLhttps://www.nponest.org
⚠ 注意 「秘密」の2つは必ず暗号化（Encrypt / Secret）で登録してください。これを入れないと会員ページがエラーになります（公開ページは鍵なしでも問題なく表示されます）。
5

独自ドメインを割り当てるwww.nponest.org で見られるようにする

  1. 作成した Pages プロジェクト → 「Custom domains」→「Set up a custom domain」
  2. www.nponest.org を追加（Step 1 が済んでいれば、住所設定とSSL証明書は自動）
  3. nponest.org（www なし）でアクセスされたら www 付きへ転送する設定（リダイレクト）を追加

※ サイトは www 付きを正式アドレスとして設定済みなので、www に揃えます。

6

公開を確認するちゃんと表示されるかチェック

  • https://www.nponest.org/ でトップページ（LP）が表示される
  • /post-parent/（親なき後ハブ）と配下の資料室・ツール・研修ページが開く
  • アドレスバーに鍵マーク（HTTPS）が出ている
  • （会員ページを動かす場合）/members/sign-in/ がエラーにならない

公開前のチェックポイント

会員ページの未完成箇所 会員ページのコミュニティ画面に、まだ仮の値（LINE_OC_URL = 'REPLACE_ME' ＝ LINEグループの招待URL未設定）が残っています。会員ページを公開するなら、実際のURLに差し替えるか、会員ページを当面は非公開にする判断が必要です。

費用の目安

  • Cloudflare Pages（サイトの公開）：無料枠で十分動きます。
  • ドメイン nponest.org：登録の実費のみ（年額。Cloudflare は手数料の上乗せなし）。
  • Supabase / Resend（会員機能・メール）：それぞれ無料枠あり。利用量に応じて従量。
おすすめの進め方 ① まず公開ページだけを www.nponest.org で公開（Step 4 を飛ばす）→ ② サイトの中身を仕上げる → ③ 会員ページの準備（鍵・LINE URL）が整ってから会員機能を有効化。段階を分けると安全で、つまずきにくいです。
