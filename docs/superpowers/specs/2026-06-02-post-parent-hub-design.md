# 「親なき後」ハブ化 設計仕様

- **日付**: 2026-06-02
- **対象**: `src/pages/post-parent.astro`（発展）、`src/pages/post-parent/library.astro`・`tools.astro`・`seminars.astro`（新規）、`src/lib/site.ts`（データ追加）
- **URL**: `/post-parent/`（親ハブ）＋ `/post-parent/library/`・`/post-parent/tools/`・`/post-parent/seminars/`
- **ブランチ**: `feature/post-parent-hub`（実装時に作成）
- **ステータス**: 設計合意済み（実装計画前）
- **先行仕様**: [`2026-05-30-post-parent-page-design.md`](./2026-05-30-post-parent-page-design.md)（現行1ページ版）を発展させる

## 1. 背景と目的

現行の「親なき後」ページ（`/post-parent/`）は **理念を語る1ページ**として完成している。本仕様は、ここに nest が蓄積した **知財（アニュアル・研修資料・支援の仕組み）を、誰でも無償で受け取れる「リソースセンター」** を重ね、さらに **ゆるい公開コミュニティの入口** を設けて、社会へ開いていくことを目的とする。

核心の想い（「お金より、遺すべきものがある」）は主役のまま維持し、その理念を**実際の知財配布という形で裏打ち**する。

## 2. 会員ページとの棲み分け（最重要）

| | 公開「親なき後」（本仕様） | 会員ページ `/members/`（既存・不可侵） |
|---|---|---|
| 対象 | **誰でも**（全国の家族・支援者・行政・他法人） | nest の**登録家族会員のみ** |
| 向き | 外向き（社会貢献・知財の無償公開） | 内向き（個別の継続支援） |
| 認証 | **不要** | ログイン必須（Supabase） |
| 中身 | 資料の無償DL・ツール紹介/デモ・研修アーカイブ・ゆるい入口 | 機関誌「巣箱」・家族会・LINE相談・my-note・admin |
| コミュニティ | **ゆるい公開入口**（メルマガ/勉強会案内、ログイン不要） | **密な内輪コミュニティ**（LINEオープンチャット） |

- **会員エリア（`src/pages/members/**`・`src/components/members/**`）には一切触れない。**
- 重複を避けるため、機能のブリッジは1本のみ：公開ハブに「もっと深く関わりたい方へ → 入会案内（`/join/`）」を控えめに置く。

## 3. スコープ（段階導入）

「ツールはデモ・体験まで実現したい」という意向を踏まえつつ、本サイトが**静的サイト（Cloudflare配信）**である制約から、段階導入とする。

### 今回のスコープ（フェーズ1：基盤）
- 案B のハブ構造を構築：親ハブ＋3子ページ＋公開コミュニティ入口セクション
- **library**：アニュアル・研修資料PDFの無償配布（その場プレビュー＋DL）
- **tools**：nest-support・エコマップを **紹介＋スクリーンショット＋動画ウォークスルー** で提示。利用したい支援者向けに導線（GitHub/問い合わせ）
- **seminars**：日本財団助成 研修会のアーカイブ（講師・テーマ・配布資料）
- **公開コミュニティ入口**：メルマガ/関心登録の軽いフォーム（DLの障壁にはしない）

### フェーズ2（次・本命：別途実装）
- **エコマップの「読み取り専用・静的デモ」**：架空1事例を **ブラウザ内（Cytoscape.js）でクライアントサイド描画**。バックエンド不要で静的サイトに載せられるため、低コストで「触れる体験」を実現する。

### フェーズ3（将来）
- フルスタックの常時稼働デモ（Neo4j＋API を別ホスティング）。継続コスト・保守・障害対応を許容できる段階で着手。

### デモ・体験に関する難所の整理（フェーズ判断の根拠）
- 本サイトは静的配信のため、DB＋バックエンドが常時動くアプリは**そのままは載らない**。本格デモは別ホスティング＝月額コスト＋保守が継続発生する。
- **nest-support は対話型**（Claude Desktop/Code ＋ MCP 前提）で、Web単体で触れるデモを作りにくい → **動画ウォークスルー＋サンプル＋（公開可なら）GitHub配布**が現実解。
- **エコマップ**は可視化部分（Cytoscape）が**ブラウザ単体で動く** → フェーズ2の静的デモが現実的に達成可能。
- 公開デモは誰でも触れる → 読み取り専用化・レート制限・定期リセットの運用が要る（フェーズ3）。
- ダミーデータで個人情報問題はクリアできるが、**北九州の地名・制度名が残ると誤解を招く**ため完全架空化する。

## 4. ページ構成

既存コンポーネント（`BaseLayout`/`Hero`/`CtaBlock`）・既存グローバルクラス（`.section`/`.section--sand`/`.container`/`.container--narrow`/`.eyebrow`/`.section__title`/`.reveal`/`.grid-3`/`.card`）を最大限流用。新規CSSは最小限。

### 4-1. 親ハブ `/post-parent/`（現行ページを発展）
1. **ヒーロー**（現行維持）
2. **理念ブロック**（現行維持＝主役）「お金より、遺すべきものがある」
3. **くらしサポートネット**（現行維持）
4. **構想：暗黙知をデジタルで継承する**（現行維持。tools ページへの導線を1本追加）
5. **【新規】リソースへの入口**：`.grid-3` の入口カード3枚 →「資料室」「ツール・しくみ」「研修アーカイブ」
6. **【新規】ゆるいつながり**：メルマガ/関心登録の案内＋フォーム導線
7. **問い合わせ動線**（現行 `CtaBlock` 維持）＋「もっと深く関わりたい方へ→入会案内」を控えめに

### 4-2. `/post-parent/library/`（資料室）
- `PageIntro` 風の見出し＋カテゴリ別一覧
  - **年次報告（アニュアル）** / **研修資料** / **その他資料**
- 各資料カード：表紙サムネ＋タイトル＋発行年＋「読む（その場プレビュー）」＋「ダウンロード」
- **PDFプレビューは sudachi ページのモーダル実装（`js-pdf-view` ＋ `#pdfModal`）をそのまま流用**する。
- データは `site.ts`（または専用データファイル）に配列で定義（`{ category, title, year, file, cover }`）。

### 4-3. `/post-parent/tools/`（ツール・しくみ）
- nest-support・エコマップを各1ブロック（`ServiceZigzag` 流用候補）で：
  - 何を解決するか / 誰向けか / どう使うか
  - スクリーンショット（＋将来は動画埋め込み）
  - 導線：エコマップ＝「デモを見る」（フェーズ2で有効化）、nest-support＝「動画を見る」「GitHub」（公開可の場合）
- 「nest の構想」（親ハブの構想セクション）と内容が連続するため、相互リンク。

### 4-4. `/post-parent/seminars/`（研修アーカイブ）
- 日本財団助成の研修会ごとに：開催年・テーマ・講師名・配布資料（PDF、library と同じプレビュー/DL）。
- 各資料に「講師の許諾を得て公開」の旨を明記。
- データは配列定義（`{ year, theme, lecturer, materials[] }`）。

### 4-5. ナビゲーション
- `navLinks` の「親なき後 → `/post-parent/`」は**現状維持**（トップレベル1項目）。
- 子ページへの回遊は**親ハブ内の入口カード**で行う（ナビにサブメニューは増やさない＝肥大化回避）。必要なら将来 children 化を検討。

## 5. 配布・運用方針

- **PDFは静的ファイル配布**：`public/docs/post-parent/`（仮）以下に配置し、Astro/Cloudflare がそのまま配信。**認証配信は不要**（会員の機関誌とは別系統）。最も簡単・無料・保守ゼロ。
- **ダウンロードは完全オープン**（メール登録などの障壁を設けない）。コミュニティ入口は**独立した任意opt-in**として分離する。
- **公開コミュニティ入口のフォーム**：本サイトは静的のため、フェーズ1では既存の問い合わせ導線と同じ方式（外部フォーム＝Google Form、または既存 contact パターン）で軽く実現する。将来 Supabase 等での会員連携はフェーズ外。
  - 補足：リポジトリには「Google Form membership intake plan」の前例（コミット `92fb052`）があり、同方式を踏襲できる。

## 6. プライバシー・コンプライアンス

- 実在の利用者・家族の個人情報、実支援データは一切掲載しない。
- ツールのデモ/スクショは**完全架空のダミーデータ**のみ。北九州固有の実在地名・施設名・人名は架空化する。
- 研修資料は**講師の配布許諾が取れているもののみ**掲載。各資料に許諾の旨を明記。
- nest-support / エコマップの紹介は概念・狙いの説明に留め、内部の実データ構造を晒さない。

## 7. コンテンツ依存（実装前に河原さんへ確認・準備が必要）

実装の前提として、以下の実素材が必要：
1. **アニュアル（年次報告）PDF** — 公開する年度分のファイル。
2. **研修資料PDF** — 講師ごとの配布資料。**配布許諾の最終確認**（誰の・どの資料か）。
3. **研修会メタ情報** — 開催年・テーマ・講師名。
4. **ツールのスクリーンショット**（最低限）と、可能なら**動画ウォークスルー**（nest-support 用）。
5. **nest-support リポジトリの公開可否** — `github.com/kazumasakawahara/nest-support` が現在 public か private か（private ならリンク前に公開設定 or 紹介のみに留める）。
6. **エコマップのGitHub公開有無** — 現在ローカルのみ（リモート未設定）。OSS配布やコードリンクを行うなら別途 GitHub 公開が必要。フェーズ1では紹介＋（フェーズ2で）静的デモのみでも成立。
7. **コミュニティ入口の受け皿** — メルマガ配信手段 or 登録フォーム（Google Form 等）の用意。

> いずれも未準備の場合、該当セクションは「準備中」プレースホルダ（placeholderLabel 等）で枠だけ用意し、素材が揃い次第差し込む方針とする（GH/post-parent と同じ運用）。

## 8. 技術メモ

- 新規ページは `group-home.astro` / 現 `post-parent.astro` と同じ骨格（`BaseLayout` + `Hero`/`PageIntro` + `section` + `CtaBlock`）。
- PDFプレビューモーダルは sudachi の実装を共通化できるか検討（重複が3箇所超なら小コンポーネント化、そうでなければ各ページにコピー）。**過剰な抽象化は避ける**。
- 資料・研修・ツールのデータは配列で `site.ts` か専用 `src/lib/post-parent.ts` に定義し、ページはループ描画。
- 新規CSSは最小限。既存グローバルクラス＋インライン `style`（既存ページと同書式）で対応。

## 9. 検証

- `npm run build` がエラーなく通り、`/post-parent/`・`/post-parent/library/`・`/post-parent/tools/`・`/post-parent/seminars/` の各ルートが生成される。
- 親ハブの入口カードから各子ページへ到達でき、各子ページから親ハブへ戻れる。
- 資料PDFが「その場プレビュー」「ダウンロード」の両方で開ける（JS無効時もDLは機能＝no-JSフォールバック）。
- 会員エリア（`/members/**`）に一切変更がない（`git diff` で確認）。
- ページ内に実在利用者・家族の個人情報・実支援データが含まれない（目視＋grep）。
- 許諾未確認の研修資料、未公開リポジトリへの実リンクが含まれない。
- レスポンシブ（375/768/1024/1280px）。
