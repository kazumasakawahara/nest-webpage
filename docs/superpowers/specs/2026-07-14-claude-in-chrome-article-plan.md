## 1. 企画の骨子

**ひとことで**: ブラウザ（Chrome）の中にAIを“住み込ませる”と、ネット上の作業——調べる・読む・入力する——をAIがその場で手伝ってくれる。その導入方法を、Chrome拡張機能の基本から、初めての人でも迷わない粒度で解説する。

**連載との関係**: 連載「AIに道具を持たせる」の**番外編（実践編）**と位置づける。連載第2回で紹介した3つの手のうち「**①外から情報を取ってくる手**」の、いちばん身近で目に見える実物が Claude in Chrome である——という接続。単体でも読み切れる構成にしつつ、冒頭または文中で連載第2回へリンクする。連載ナビ（全5回）には**加えない**（完結済みのため）。公開後、第2回の本文末尾に「関連記事」として相互リンクを追記するのは任意（河原さん判断）。

**読者層**: 既存読者のうち「ChatGPTやGeminiのチャットは使ったことがある」層。連載より半歩やさしく——拡張機能という言葉から説明する。中級者編バッジは付けず、通常記事として出すことを推奨（連載で背伸びした読者への“ごほうび回”）。

いずれも「住み込み」の比喩を核にする。この比喩は本文の背骨にもなる（§4）。

## 3. 記事の核となるたとえ話

これまでのAIは「**電話口の相談員**」（連載第2回）や「**別室にいる助手**」——聞けば答えてくれるが、こちらの画面は見えない。Claude in Chrome を入れると、AIが**ブラウザの中に住み込む**。つまり：

- 今あなたが見ているページを、**同じ画面を見ながら**手伝ってくれる（読む・要約する・explainする）
- 頼めば**代わりに手を動かす**（ページを開く、フォームに入力する、必要な情報を拾い集める）
- 住み込みだから、**呼べばすぐ隣にいる**（サイドパネル／ピン留めしたアイコンから）

「チャット画面にコピペして持っていく」から「作業している現場に来てもらう」への転換、と整理する。

## 4. 構成案（見出しレベル）

1. **導入**: 現場の場面から。WAM NETや自治体サイトを開いて「この長いページ、どこを読めばいいの…」——それ、隣で一緒に見てくれるAIがいたら？
2. **そもそも「拡張機能」ってなに？**: Chromeに後づけできる小さな道具。スマホのアプリの追加と同じ、という説明。Chromeウェブストア＝公式のアプリ屋さん。「提供元を確認して入れる」という安全習慣もここで。
3. **Claude in Chrome を入れてみよう**（手順・スクショ中心）:
   - Chromeウェブストアで検索 → 提供元が Anthropic であることを確認 →「Chromeに追加」
   - Claudeアカウントでサインイン（**有料プラン対象**である旨を明記。§6の事実確認リスト参照）
   - **ピン留め**: パズルピースのアイコン → ピンのマークを押すと、住み込みAIの“呼び鈴”がいつも見える場所に。ピン留めの意味（隠れてしまわない・ワンクリックで呼べる）を丁寧に。
4. **住み込みAIに、何を頼めるか**（現場ユースケース）:
   - **主役の実演（実機スクショあり・§5）**: WAM NETを開いて「北九州市小倉北区にある放課後等デイサービス事業所を一覧表示して」と一言 → Claudeが**作業プランを提示して承認を待つ** → 承認すると検索・一覧化まで実行。連載第3回で「道具を持ったAIのWeb巡回」として説明したことが、**誰でも入れられる拡張機能でここまでできる**という橋渡しになる
   - 開いているページの要約・解説（「このページ、うちの利用者さんに関係ある部分だけ教えて」）
   - 複数タブを見比べての整理（「開いている3つの事業所ページ、違いを表にして」）
5. **気をつけたいこと**（必須callout）:
   - サイトごとの許可制・大事な操作（送信・購入など）の前には確認が入る、という設計を紹介
   - **個人情報を扱う画面（支援記録システム・オンラインバンキング等）では使わない／許可しない**。事業所ルールの確認
   - AIの操作は必ず人が見届ける（「任せっぱなし」にしない）
6. **GeminiやChatGPTでも、できるの？**（フェアネス回。3社3様のアプローチを対比図で）:
   - Claude → **拡張機能**として今のChromeに住み込む
   - Gemini → **Chromeに標準搭載**されつつある（Googleが家主なので、家に最初から住んでいる）
   - ChatGPT → **Atlasという専用ブラウザ**／ChatGPT内のエージェント機能（家ごと新築する方式）
   - 「どれが偉いかではなく、住まわせ方の違い」という整理。連載第1回のフェアネス原則を踏襲
7. **豆知識: computer use ― ブラウザの外へ**: そもそもAIが「画面を見て、マウスとキーボードを操作する」能力を computer use（コンピュータ操作）と呼ぶ。Claude in Chrome はそれをブラウザという安全な範囲に絞ったもの。技術はすでに「パソコン全体の操作」まで広がりつつある——という半歩先の展望で締める。
8. **締め**: 連載の言葉で回収——「これも立派な、AIに道具を持たせるの第一歩」。第2回へのリンク＋お問い合わせ定型文。

## 6. 事実確認リスト（執筆前に最新情報で裏取りすること）

2026-07-14 時点の下調べでは以下のとおり。**UI名称・プラン要件は変わりやすいので、執筆時と公開直前に必ず再確認**する。

| 項目 | 下調べの結果（要再確認） |
|---|---|
| 提供形態 | Chrome拡張機能（Chromeウェブストア、提供元 Anthropic） |
| 対象プラン | 有料プラン（Pro/Max/Team/Enterprise）。無料プランは対象外 |
| 日本語 | 指示・応答とも日本語可 |
| 安全設計 | サイトごとの許可制、送信・購入等の前に確認プロンプト、高リスクサイトのブロック |
| Gemini | Chrome標準搭載の方向（Ask Gemini等）。地域・プランにより提供状況が異なる |
| ChatGPT | Atlas（専用ブラウザ）およびChatGPT内エージェント機能 |
| computer use | Anthropicが公開している「画面を見て操作する」能力の呼称。ブラウザ版がClaude in Chrome |

**実機で確認済みの事実（2026-07-14・河原さんのスクショより）**: サイドパネル型のチャットUI／モデル選択表示（例: Sonnet 5）／「実行前に確認」の設定表示／作業前に**プラン（従うべきアプローチ）を提示して承認を求める**フロー／対象サイト単位の許可（例: wam.go.jp）／実行中はページ側に「Stop Claude」ボタン。——これらは一次情報として本文の記述に使ってよい（Web記事の記述より優先）。

注意: 拡張機能のセキュリティは進化の途中でもある（脆弱性の報告と修正が続いている）。記事では個別の脆弱性名には触れず、「大事な画面では使わない」「確認を待つ」という**利用者側の習慣**に落とし込む。
