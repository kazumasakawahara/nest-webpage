1.何をつくるか（概要）

現状、入会申込は HTML 形式の申込書を印刷 → 手書き → 郵送/持参/FAX という紙ベースの一択です。これに Google Form 経由の電子申込ルートを追加し、QR コードでスマホから直接アクセスできるようにします。

申込内容は Google Spreadsheet に自動集約され、事務局がブラウザだけで管理できる状態を目指します。

期待される効果
  • スマホ世代（30〜50代の家族層）が申込しやすくなる
  • 事務局の手入力作業が減る（Spreadsheet に最初からデータが揃う）
  • 申込件数・属性のデータが集計可能になる（広報の振り返りに使える）
注意点
  • 紙ルートも残す：高齢者・スマホ非保有者にも届けるため
  • 個人情報の越境：Google Forms = 米国サーバ（後述）

2.全体アーキテクチャの3レベル

自動化の度合いで3パターン用意できます。手間と機能のトレードオフです。

推奨
Level 1
Form + Sheet 単独
Google Form の標準機能だけで完結。最小工数。
○ 30分でセットアップ可
○ ノーコード
○ 月額0円
○ 林代表/河原さんで保守できる
× nest 会員DBへの登録は手動コピペ
× 確認メール送信は手動 or Form の標準機能のみ
Level 2
+ Apps Script 半自動
Google Apps Script で通知・ステータス管理を自動化。
○ 受付メール自動送信（事務局＆申込者）
○ ステータス列を Spreadsheet 側で管理
○ 月額0円
○ JS の基本知識で書ける（20〜50行）
× Apps Script の保守が必要
× nest 会員DBへの登録は依然手動
Level 3
Supabase 自動連携
Form → Apps Script → Supabase API で会員自動登録。
○ 完全自動化、事務局の作業ほぼゼロ
○ ようこそメール（Resend）も自動
× 会費納付確認の手動判定が依然残る
× エラーハンドリングが複雑
× Apps Script に Supabase の secret_key を持たせる必要あり
× Phase 2 マターのスコープ
推奨：Level 1 で開始し、運用が定着したら Level 2 に格上げ。Level 3 は Phase 2 で議論。
Level 1 でも「事務局が新規申込を見落とす」リスクは Google Form 標準の「回答時にメール通知」機能で十分カバーできます。

3.Google Form のフィールド仕様案

HTML 紙申込書の項目と整合させ、Google Form の機能（選択肢、必須/任意、条件分岐）を活用します。

  • 項目名フォーム部品必須/任意
  • 会員区分ラジオ 正会員 / 賛助会員（個人）/ 賛助会員（団体）必須
  • お名前（団体名）短文必須
  • お名前（ふりがな）短文必須
  • メールアドレスメール 自動バリデーション必須
  • 電話番号短文 正規表現バリデーション可必須
  • 郵便番号短文必須
  • 住所短文必須
  • 生年月日日付任意
  • ご職業・ご所属短文任意
  • nest を知ったきっかけ複数選択 Webサイト / 知人紹介 / イベント / SNS / その他任意
  • 入会の動機・nest への期待長文任意
  • ボランティア参加意向ラジオ 希望する / 応相談 / 希望しない任意
  • 緊急連絡先（任意）短文 氏名・続柄・電話任意
  • 個人情報取り扱いの同意チェック 同意する必須
  • その他のご要望・質問長文任意
Google Form の「セクション分岐」機能で、賛助会員（団体）を選択した場合のみ「団体担当者名」「団体住所」を表示する、といった出し分けも可能です。Level 1 では使わず、シンプルに1ページで揃えるのが推奨。

4.QR コードの生成・配置

4-1. 生成方法（3つの候補）
推奨
A. 無料ツールで PNG 生成
Google Chart API や qr-code-generator.com で 1分でPNG生成。
○ 最簡単、依存ゼロ
○ Form URL は変わらないので一度作れば終わり
○ 印刷物にも流用可
× Form URL を変更したら再生成必要
B. ビルド時 SVG 生成
qrcode npm パッケージで build 時に自動生成。
○ URL を環境変数で管理できる
○ SVG なので印刷時もシャープ
× ビルドパイプライン複雑化
× Phase 1 のスコープ外
C. ブランド入り QR
中央に nest ロゴ、テラコッタ色を使った装飾 QR。
○ ブランド統一感
○ 印刷物にも映える
× qr-code-monkey.com 等で手動デザイン
× 装飾しすぎるとスキャン失敗率↑（要テスト）
推奨：A で開始 → 余裕があれば C にアップグレード。
生成済み PNG は public/images/qr/membership-form.png に配置し、/join/ ページに <img> で表示します。
4-2. 推奨サイズ・配置
用途サイズ備考
Web ページ（/join/）200×200 pxretina 用に 400×400 PNG を 200×200 表示
印刷物（チラシ等）2cm 角以上解像度 300dpi 推奨
会員カード裏面（将来）1.5cm 角余白（quiet zone）4モジュール必要
4-3. 補足：URL 短縮の検討

Google Form の URL は長い（例：https://docs.google.com/forms/d/e/1FAIpQLSc.../viewform）ため、QR コードのデータ密度が高くなり、印刷時に読み取りにくくなる可能性があります。

選択肢：

  • そのまま使う：QR は問題なくスキャン可。URL コピペで共有も可
  • 独自ドメインで短縮 URL：例 nponest.org/apply → Form にリダイレクト。/join/ ページ内に短縮表記、QR は独自ドメイン URL を埋め込み。Cloudflare Pages の Redirect ルールで実装可能
  • bit.ly 等の短縮サービス：依存サービス追加のため非推奨

6.個人情報保護の観点

HTML 申込書（紙）と Google Form（電子）では、個人情報の流れが大きく異なります。

HTML 申込書（紙） Google Form
データの流れ 本人 → 紙 → 事務局 本人 → Google サーバ（米国）→ Spreadsheet
越境移転 なし あり（米国）
本人同意 申込書面の同意欄 Form 内に明示的な同意項目が必須
同意取り消し対応 紙の破棄 Spreadsheet 行の削除 + Google アカウント側からの削除依頼
セキュリティ 物理保管（金庫等） Google のセキュリティ + 事務局アカウントの2段階認証必須
個情法上の対応：
Form の最初か最後に、「ご記入いただく個人情報は Google 社（米国）のサーバを経由します。同意の上ご入力ください」という記載＋同意チェックを必須に。
現行の /members/privacy ページの方針（Supabase 東京リージョン、越境移転なし）と一貫しないので、Form の利用にあたって 個人情報取扱い説明文の追記が必要です。
推奨される同意文（叩き台）
本フォームに入力いただいた個人情報は、Google LLC（米国）が運営する
Google Forms のサーバを経由して NPO法人 nest が受信・保管します。

利用目的：会員管理、会報誌の送付、行事のご案内
保管期間：会員として在籍している期間
第三者提供：法令に基づく場合を除き、行いません

紙ベースでの申込をご希望の方は、以下のURLから入会申込書（PDF）を
ダウンロードしてご利用ください。
https://www.nponest.org/pdfs/forms/membership-application.html

9.林代表に確認したいこと

本計画を進める前に、林代表との合意が必要な事項です：

Q1. 個人情報の Google 越境移転について

Google Forms を利用すると、申込者の個人情報が Google LLC（米国）のサーバを経由します。 これは /members/privacy でうたっている「越境移転なし」と若干食い違います。

選択肢：

  • (A) 許容する：Form 上に明示的な同意文を出すことで対応
  • (B) 許容しない：紙ルートのみで運用継続
Q2. 自動化レベル（§2）

Level 1 / 2 / 3 のどれで始めるか。推奨は Level 1（Form + Sheet）。

Q3. /join/ ページ動線（§7）

α / β / γ のどれを採用するか。推奨は β（並列）。

Q4. 会費体系

正会員 / 賛助会員（個人）/ 賛助会員（団体）の年会費額。 HTML 申込書（§3）でも未確定。

Q5. 会費納付方法

銀行振込 / 郵便振替 / 現金持参 / オンライン決済の対応有無。 Form のサンクスメッセージで案内する文言が決まる。

Q6. Form / Spreadsheet のオーナー

Google アカウント所有者は誰になるか。事務局共用アカウント／法人代表アカウント／事務局スタッフ個人アカウントなど。 退職時のアカウント引継ぎリスクを考慮。

Q7. nest 会員 DB との連携時期

Phase 1 リリース後すぐ Level 2/3 に進むか、 まずは Level 1 で運用してから判断するか。
