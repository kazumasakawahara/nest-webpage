## 1. 背景と目的

記事「AIと"申し送りノート"を持つ」は Claude Desktop での共有フォルダ接続を主題にしているが、読者の多くは Gemini や ChatGPT のユーザーでもある。連載の先行例（Claude in Chrome 記事の「GeminiやChatGPTでも、できるの？」節・3社3様の図）と同じ精神で、**他のAIでの実現手順、なければ正直な代替手段**を提示する。

## 2. リサーチフェーズ（実装前に必須）

### 2-1. 事実の骨子（要再検証）

**ChatGPT（OpenAI）**
- ChatGPT デスクトップアプリに、設定画面から MCP サーバーを追加する UI がある（Settings → MCP servers → Add server。STDIO / Streamable HTTP を選択）
- ChatGPT デスクトップアプリ・Codex CLI・IDE 拡張は同一の設定（`~/.codex/config.toml` の `[mcp_servers.*]`）を共有する。一度設定すればどのクライアントからも使える
- filesystem サーバーは `@modelcontextprotocol/server-filesystem`（Claude と同じもの）が使える。`codex mcp add` コマンドでも追加可
- → 記事上の扱い: **「ChatGPT でもできる」**。デスクトップアプリの設定 UI ルートを主に紹介（読者層に合う）。Codex CLI は「その先」の一言に留める

**Gemini（Google）**
- 一般向け Gemini アプリ（Web / スマホ）には、ローカルフォルダを直接つなぐ公式手段が**ない**
- MCP 対応があるのは **Gemini CLI**（`~/.gemini/settings.json` の mcpServers、または `gemini mcp add`。`/mcp` で接続確認）。ただし黒い画面のツールであり、本記事の読者層には遠い
- → 記事上の扱い: **正直に「アプリ単体ではできない」＋代替2本立て**
  - 代替①（本命）: **Google ドライブを"共有の棚"にする**。Gemini は Google ドライブと公式連携しており、Drive 上のフォルダ・ファイルを読める。「棚をパソコンの中ではなく、Google ドライブに置く」という発想の転換として紹介。申し送りノートの比喩がそのまま使える
  - 代替②（一言だけ）: Gemini CLI ならローカルフォルダも可能（上級者向け、と明示）
- 注意: Drive 連携の書き込み可否（Gemini が Drive にファイルを**書ける**か、読むだけか）は要検証。読むだけなら「申し送りを書くのは人間、読むのはAI」という半分の実現になる。事実に合わせて正確に書くこと

## 7. 設計メモ（2026-07-16 Claude Code 追記）

### 7-1. 一次情報での再検証結果

**確認できた事実（骨子どおり）**

- **ChatGPT**: OpenAI 公式 Learn ドキュメント（learn.chatgpt.com/docs/extend/mcp）で確認。デスクトップアプリの **Settings → MCP servers → Add server**（STDIO / Streamable HTTP を選択）。設定は `~/.codex/config.toml` の `[mcp_servers.*]` に保存され、**デスクトップアプリ・Codex CLI・IDE拡張で共有**（"The ChatGPT desktop app, Codex CLI, and IDE extension share this configuration"）
- **Gemini CLI**: 公式 docs（geminicli.com/docs/tools/mcp-server ほか）で確認。`~/.gemini/settings.json` の mcpServers / `gemini mcp add` / `/mcp` で接続確認。骨子どおり
- **Gemini アプリ×ドライブ（代替①）**: 公式ヘルプ（support.google.com/gemini/answer/15229592）で確認。**@Googleドライブでファイルの検索・読み取り・要約が可能**。仕事用・学校用アカウントは管理者がアプリ連携を有効化する必要あり

**変わっていた・注意が必要な点**

1. **Gemini のドライブ書き込みは「できません」と公式に明記**（同ヘルプ：ファイルの作成・下書き・削除、フォルダ作成・コンテンツ移動は不可）。→ 骨子 §2-1 の要検証事項の答えは「**読み取り専用**」。代替①は「読む側半分の実現」（申し送りを書くのは人間、読むのはAI）として正直に書く
2. **ChatGPT の Mac 版で追加できるかは断定不能**: 2025年秋頃の記事（4sysops等）には「Mac アプリでは MCP 追加不可（Developer mode は Windows/Web のみ・リモートサーバーのみ）」という情報が流通。現行の公式ドキュメントにはプラットフォーム制限の記載がない（新しい MCP servers 設定と旧 Developer mode コネクタは別機能とみられる）。→ 記事では **OS 別の断定を避け**、メニュー名の紹介＋「迷ったらAI本人に聞く」原則で吸収
3. filesystem サーバー（`@modelcontextprotocol/server-filesystem`）が ChatGPT で使える旨の**公式の明示例は見つからず**（公式は Context7 等を例示）。MCP が共通規格である以上 STDIO サーバーとして登録可能だが、記事では「**同じ規格（MCP）だから、Claude で使ったのと同じ道具が挿さる**」という規格レベルの記述に留め、手順の細部は書かない
4. 周辺事実: ドライブ側パネルの「Gemini in Google ドライブ」は対象プラン限定で、2026-04-20 以降サイドパネルからのファイル作成・移動機能が廃止された → 記事では踏み込まない（混乱防止）

### 7-2. 掲載形態の判断

**案A（記事本体に節を追加）を採用**。確認できた情報で書ける分量は、ChatGPT 2〜3ステップ＋Gemini 代替提示で 700〜900字に収まり、独立記事にするほどの厚みはない（手順の細部を書かない方針のため）。節の位置は指示書どおり「その先の景色」と「気をつけたいこと」の間。
